# 人事管理

## 背景

宿泊施設の運営には様々な役割を持つスタッフが関わるため、スタッフ情報と権限の適切な管理が不可欠である。スタッフの役割に応じた権限設定により、システムの安全性を確保しながら業務効率を向上させる必要がある。

## 目的

- スタッフの入退社に伴う情報管理を効率化する
- 役割に応じた適切な権限設定により、情報セキュリティを確保する
- パスワード管理機能により、アカウントセキュリティを維持する

## 主要アクター

- 人事担当者

## 業務概要

人事管理では、スタッフの登録・更新・削除、パスワードのリセット、権限ロールの割り当て・変更・確認を行う。スタッフ情報には名前、メールアドレス、電話番号、権限ロールが含まれる。権限ロールはAdmin、Reservation Manager、Front Desk、Housekeeping、Accounting、Owner View Onlyの6種類があり、それぞれ利用可能な機能と表示項目が制限される。

## 要求

### 人事担当者
スタッフ情報の登録・更新・削除、権限ロールの割り当て、パスワードリセットを行う

## 機能

### スタッフを管理する
- スタッフを登録する
  - スタッフを登録する（画面：スタッフ登録画面、アクター：人事担当者）
  - 新規スタッフの情報を登録する
- スタッフを更新する
  - スタッフを更新する（画面：スタッフ更新画面、アクター：人事担当者）
  - 既存スタッフの情報を変更する
- スタッフを削除する
  - スタッフを削除する（画面：スタッフ削除画面、アクター：人事担当者）
  - 退職などで不要になったスタッフ情報を削除する
- パスワードをリセットする
  - パスワードをリセットする（画面：パスワードリセット画面、アクター：人事担当者）
  - スタッフのパスワードを再設定する

### 権限を管理する
- 権限を割り当てる
  - 権限を割り当てる（画面：権限割り当てる画面、アクター：人事担当者）
  - スタッフにロールを設定する
- 権限を変更する
  - 権限を変更する（画面：権限変更画面、アクター：人事担当者）
  - スタッフのロールを変更する
- 権限を確認する
  - 権限を検索する（画面：権限検索画面、アクター：人事担当者）
  - スタッフの権限設定を確認する

## ビジネスルール

### 権限による機能制限
- コンテキスト：人事管理
- 説明：スタッフの権限ロールに基づいて、利用可能な機能と表示項目を制限する
- 関連バリエーション：権限ロール（Admin, Reservation Manager, Front Desk, Housekeeping, Accounting, Owner View Only）
- 適用ルール：全ての権限ロールに対して自動適用

## ビジネスパラメーター

### 権限ロール
- 値：Admin、Reservation Manager、Front Desk、Housekeeping、Accounting、Owner View Only
- 説明：スタッフの役割を表し、利用可能な機能や表示項目の制限に使用する
- 各ロールの権限：
  - Admin：全機能へのアクセス権限
  - Reservation Manager：予約管理機能へのアクセス権限
  - Front Desk：フロント業務機能へのアクセス権限
  - Housekeeping：清掃管理機能へのアクセス権限
  - Accounting：経理・売上分析機能へのアクセス権限
  - Owner View Only：閲覧のみの権限
